# 🔧 地图加载问题诊断指南

## 问题现象
司机端显示"腾讯地图 API 加载失败"

## 可能原因及解决方案

### 1️⃣ 网络无法访问腾讯地图服务器

**检查方法**:
```bash
curl -I "https://map.qq.com/api/js?v=2.exp"
```

**解决方案**:
- 检查服务器网络连接
- 确认防火墙未阻止 map.qq.com
- 如果是国内服务器，应该可以正常访问

### 2️⃣ 浏览器扩展拦截（广告拦截器）

**症状**: 控制台显示脚本被阻止

**解决方案**:
1. 关闭广告拦截器（AdBlock、uBlock Origin 等）
2. 将站点加入白名单
3. 使用无痕模式测试

### 3️⃣ HTTPS 混合内容问题

**症状**: 通过 HTTP 访问页面，但地图 API 是 HTTPS

**解决方案**:
- 使用 HTTPS 访问站点
- 或使用 `http://` 协议访问地图 API（不推荐）

### 4️⃣ API Key 无效或被限制

**症状**: 控制台显示 "Invalid Key" 或 "Referer not allowed"

**解决方案**:
1. 登录 [腾讯地图开放平台](https://lbs.qq.com/)
2. 检查 Key 是否有效
3. 检查 Referer 白名单设置
4. 检查配额是否用完

**当前使用的 Key**: `67MBZ-EF6RT-THAX4-VEGBL-7AYMJ-LGBXX`

### 5️⃣ CSP（内容安全策略）限制

**症状**: 控制台显示 "Refused to load script from..."

**解决方案**:
检查服务器响应头中的 `Content-Security-Policy`，确保包含：
```
script-src 'self' https://map.qq.com https://3gimg.qq.com
```

### 6️⃣ 浏览器不支持或版本过旧

**检查方法**:
```javascript
console.log(navigator.userAgent);
```

**解决方案**:
- 使用现代浏览器（Chrome 80+、Firefox 75+、Safari 13+）
- 更新浏览器到最新版本

## 🧪 诊断步骤

### 步骤 1: 访问诊断页面
打开 `http://localhost:3000/test-map-load.html`

查看输出结果，确定是哪一步失败。

### 步骤 2: 检查浏览器控制台

按 F12 打开开发者工具，查看 Console 标签：

**正常情况**:
```
✅ 腾讯地图 API 加载成功
qq 对象: object
qq.maps 对象: object
```

**失败情况**:
```
❌ 腾讯地图 API 加载失败
错误事件：{"type":"error","target":"https://map.qq.com/api/js?..."}
```

### 步骤 3: 检查 Network 标签

在开发者工具的 Network 标签中：
- 过滤 "map.qq.com"
- 查看请求状态码
- 检查响应内容

**正常**: 状态码 200，响应是 JavaScript 代码

**失败**: 
- 状态码 (failed) - 网络问题
- 状态码 403 - API Key 问题
- 状态码 0 - 被拦截

### 步骤 4: 测试外部地图

即使地图加载失败，司机仍可使用"打开外部导航"按钮：
- 点击订单详情中的"🧭 打开外部导航"
- 会在手机地图 APP 中打开路线

## ✅ 降级方案

即使地图无法加载，司机端仍可正常使用：

1. **查看订单信息** - 文字信息正常显示
2. **确认/拒绝任务** - 功能正常
3. **更新状态** - 到达现场、完成任务等正常
4. **外部导航** - 点击按钮打开手机地图 APP

## 📱 手机端特殊处理

如果在手机上使用：

1. **确保允许定位权限**
   - iOS: 设置 → 隐私 → 定位服务
   - Android: 设置 → 应用权限 → 位置信息

2. **使用 HTTPS**
   - 手机浏览器对非 HTTPS 站点的定位限制更严格

3. **关闭省流量模式**
   - 可能阻止地图资源加载

## 🔧 临时解决方案

如果问题持续，可以：

1. **改用高德地图**（需要申请 Key）
2. **完全移除地图依赖**（仅显示文字地址）
3. **使用静态地图图片**（不推荐，交互性差）

1. 浏览器类型和版本
2. 访问地址（HTTP 还是 HTTPS）
3. 控制台完整错误信息
4. Network 标签中地图请求的状态码
